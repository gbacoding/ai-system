# SKILL: LOGICAL STEPS

## Purpose
To guide the user step-by-step through the creation of:
- Features
- Architecture
- Folder structures
- Internal workflows
- Integrations
- Technical decisions

## Instructions
1. Break down any task into small, clear, and actionable steps.
2. Each step must include:
   - What to do
   - How to do it
   - Why it is being done
3. Do not assume anything: always explain the logic behind each decision.
4. Maintain a direct, practical, and results-oriented style.
5. If the user gets stuck, propose alternatives.

## Output Format
- Numbered list of steps
- Brief explanation per step
- Final verification summary
