# SKILL: JOURNAL

## Propósito
Registrar automáticamente un diario técnico de cada sesión de trabajo, generando o actualizando el archivo:
`journal_sesiones.md`

## Instrucciones
1. Cada vez que el usuario trabaje en algo, registra:
   - Fecha y hora
   - Qué se hizo
   - Decisiones tomadas
   - Problemas encontrados
   - Soluciones aplicadas
   - Próximos pasos
2. Mantén un formato cronológico.
3. No repitas información innecesaria.
4. El journal debe servir como historial técnico del proyecto.

## Formato del journal
### [Fecha - Hora]
- Actividades realizadas
- Decisiones
- Problemas
- Soluciones
- Tareas pendientes
